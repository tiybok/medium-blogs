Create High Availability Architecture with AWS CLI

The Architecture Includes Web-App that is using EC2,EBC,S3,Cloud-Front for Less Latency and Faster Content Delivery

Create High Availability Architecture with AWS CLI

The architecture includes-

Web-server configured on EC2 Instance.

Static objects used in code such as pictures stored in S3.

Setting up Content Delivery Network using Cloud-Front and using the origin domain as S3 bucket.

Document Root (/var/www/html) made persistent by mounting on EBS Block Device.

Finally place the Cloud Front URL on the web-app (webpage) code for security and low latency.

Create IAM User

To login to AWS account from the AWS CLI need the access and secret key.To get the access and secret key from IAM services need to add user and given Access type programmatic access(AWS CLI) and AWS Management console access(AWS WebUI).

Attach PowerUserAccess Policy to the IAM User

Download the .csv file

Configure it with terminal using command

aws configure

Create a key-pair

aws ec2 create-key-pair --key-name arth_key

Confirmation of Key Creation on AWS Web Console

Create a security group

The command to create a security group is:

aws ec2 create-security-group --description “everyone allowed”      --group-name ABC

Confirmation of Security Group on AWS Web Console

Add ingress rule on port 80 for security group

aws ec2 authorize-security-group-ingress --group-id                sg-02321675dd9ad92 --protocol tcp --port 80 --cidr 0.0.0.0/0

As we can see port 80 attached to Security Group.

Description of security group using AWS CLI

aws ec2 describe-security-groups --group-id sg-02321e675d9d9ad92

Launch an instance using the above created key pair and security group

aws ec2 run-instances --security-group-ids sg-02321e675d9d9ad92     --instance-type t2.micro --count 1 --image-id ami-0e306788ff2473ccb   --key-name arth_key

Confirmation of Creation of EC2-Instance on AWS Web-Console

Create an EBS volume of 1 GB and attach it to the instance created above

aws ec2 create-volume --avaiability-zone ap-south-1a --volume-type gp2 --size 1

Confirmed 1 GiB in Volumes on AWS Web-Console

Attach created EBS volume of 1 GiB to an instance

aws ec2 attach-volume --instance-id i-08616a2555caa235d1 --volume-id vol-05a47ec79804e3f55 --device /dev/sdh

Confirmation of EBS Volume of 1 GiB EC2-Instance Web-Console under Storage Section

Confirmation in Volume Used in Volumes on EC2-Instance Web-Console

Confirmation of Storage in-use from AWS CLI

Create S3 bucket from command

aws s3 mb s3://myarth-bucket --region ap-south-1

Confirmation S3 bucket on AWS Web-Console

Coping Image (object) in S3 bucket:

aws s3 cp Sample.png s3://myarth-bucket

Confirmation of Image (object) uploaded on AWS Web-Console

Image is uploaded

Now create cloud-front web distribution for faster content delivery across the world from any of the location using origin as s3.

aws cloudfront create-distribution --origin-domain-name myarth-bucket.s3.amazonaws.com

Go inside EC2-Instance CLI

fdisk -l
fdisk  /dev/xvdh
n option gives
p option gives me partition name with size
w option gives

Now formatted it with ext4 format type.

mkfs.ext4 /dev/xvdh1

Now i mounted this partition with /var/ww/html/ folder to make web server data persistent

mount /var/www/htm^Cweb/  /dev/xvdh1

then

mount /dev/xvdh1/  /var/www/html/web/

Now click on edit public access settings of bucket

Allow all public access so all clients can access it form anywhere.

Now object is made public

Go inside EC2-Instance

Install httpd Inside Instance

yum install httpd -y

Now i started httpd services using systemctl

systemctl start httpd

Disable selinux security to allow clients

setenforce 0

Command to see all mounted partitions and /dev/xvdh1 is mounted to /var/www/html/ folder.

dh -h

Put code in web.html inside /var/www/html

cat web.html

Go to public-ip/web.html to check the web-app (webpage)

https://13.233.178.205/web.html

Web-App (webpage) that is using EC2, EBS, S3, Cloud-Front for Less Latency and Faster Content Delivery is Completed.
