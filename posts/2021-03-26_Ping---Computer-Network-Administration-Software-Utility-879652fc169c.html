Ping : Computer Network Administration Software Utility

Create a Setup so that you can ping google but not able to ping Facebook from same system.

Ping is a computer network administration software utility used to test the reachability of a host on an Internet Protocol (IP) network. It is available for virtually all operating systems that have networking capability, including most embedded network administration software.

Ping measures the round-trip time for messages sent from the originating host to a destination computer that are echoed back to the source. The name comes from active sonar terminology that sends a pulse of sound and listens for the echo to detect objects under water.

Ping operates by sending Internet Control Message Protocol (ICMP) echo request packets to the target host and waiting for an ICMP echo reply. The program reports errors, packet loss, and a statistical summary of the results, typically including the minimum, maximum, the mean round-trip times, and standard deviation of the mean.

The command-line options of the ping utility and its output vary between the numerous implementations. Options may include the size of the payload, count of tests, limits for the number of network hops (TTL) that probes traverse, interval between the requests and time to wait for a response. Many systems provide a companion utility ping6, for testing on Internet Protocol version 6 (IPv6) networks, which implement ICMPv6.

Let Check If System Is Ping-able To Google And Facebook

To View the Routing table of the System :

route -n

Checking connection with google and facebook before configuring the Routing table:

Check connection with Google

ping -c 10 google.com

Google is ping-able at the moment

nslookup google.com

Check connection with Facebook

ping -c 10 facebook.com

Facebook is also ping-able at the moment

nslookup facebook.com

We are able to ping to all the IPs due to the presence of 0.0.0.0 IP in the routing table. It is able to ping to all the IP’s

Now starting the configuration as system that can ping google but not able to ping Facebook from same system

Deleting the default gateway of the system :

route del -net 0.0.0.0

After this , if we try to ping , it will not work

Now we only add the IP address of google to the Routing Table , which we want to ping using command :

nslookup google.com

Command to Add new IP to Routing Table :

route add -net 172.217.160.0/24 gw 192.168.255.1 enp0s3

After this , we can ping to google and when we ping facebook it will not work.

Hence the system configuration is complete.

If We Want To Reset The Routing Table To Normal And Make Both (Google And Facebook) Ping-able On Same System

Then first delete the added new IP by using this command

route del -net 172.217.160.0/24 gw 192.168.255.1 enp0s3

Now add new IP to Routing Table :

route add -net 0.0.0.0 gw 192.168.255.1 enp0s3

Now we are able to ping to all the IPs due to the presence of 0.0.0.0 IP in the routing table. It is able to ping to all the IP’s.Both Google and Facebook are ping-able as normal.

ping -c 5google.com
ping -c 5 facebook.com

Conclusion

Ping is a command-line network utility that allows you to test the IP-level connectivity of a given host on the network.
